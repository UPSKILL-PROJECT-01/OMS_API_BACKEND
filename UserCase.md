# UC1 - Registar utilizador
```
Quando um utilizador não registado insere seus dados (credenciais de acesso, nome e email) no sistema, este realiza uma validação inicial dos dados e solicita uma confirmação ao utilizador. Após a confirmação dos dados inseridos, o sistema conclui o registo, armazena as informações e notifica o utilizador sobre o sucesso da operação.
```


# UC2 - Consutar surtos, recomendações e informações gerais

``` 
A API oferece um amplo acesso informacional ao utilizador, permitindo-lhe consultar dados essenciais sobre surtos de saúde de forma rápida e detalhada. Independentemente de estar registado, o utilizador pode acessar uma variedade de endpoints de leitura para obter informações atualizadas sobre surtos ativos por país, recomendações específicas para cada localidade e surto, além de dados históricos sobre surtos passados. Essa estrutura proporciona uma visão abrangente dos dados não sensíveis da base, oferecendo ao utilizador flexibilidade máxima para explorar e utilizar a informação de maneira livre e acessível.

Além disso, a API também disponibiliza endpoints para levantar informações sobre os países e zonas geográficas presentes na base de dados, permitindo ao utilizador realizar estudos mais detalhados e personalizados, com um entendimento completo do contexto geográfico dos surtos. Combinando dados geográficos e epidemiológicos, a API torna-se uma ferramenta robusta para análises e pesquisas aprofundadas, proporcionando um elevado poder informacional.
```
